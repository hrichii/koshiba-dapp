# 寺院×ICPアプリケーション：デジタル檀家サービス

## サービスコンセプト

![サービスコンセプト図](/src/README-img/Service_title.png)

「檀家が納得感する透明性ある寺運営」と「住職にとって効率的で安定した収入」を両立する新しい仕組みです。従来の檀家制度の課題を解決するだけでなく、寺院と檀家の関係性を密接なものへと再構築し、コミュニティとしての絆を強化します。

### コンセプトの核となる要素

- **グレード制デジタル檀家証の発行**
  - 檀家（だんか）に対してグレード別の「デジタル檀家証」を発行
  - 所属や貢献度を可視化し、寺院との関係性を明確に証明
  - 各グレードは単なる金額区分ではなく、寺院への関わり方・支援の形を表現しており、グレードに応じて受けられるサービスが変化

- **オンチェーン管理による資金透明化**
  - 寄付やお布施の使途をブロックチェーン上で記録・公開
  - 集められた資金がどのように使われたかを誰でも確認可能
  - 「お金の流れ」を透明化することで、檀家の信頼を獲得

- **コミュニティ型の寺院運営への参加**
  - 檀家はグレードに応じた特典や役割を獲得
  - 寺院の意思決定に投票や意見表明を通じて参画
  - 檀家がより主体的に寺運営に関わっていき「みんなで守り、育てる寺院」という新しい関係性の構築

### 既存の檀家制度からの進化

現在の檀家制度では、世襲的な継承や地理的な制約、不透明な資金運用などの課題があります。本サービスは、これらの課題を技術で解決しながらも、寺院の伝統的価値や文化は守る「進化型檀家システム」を目指します。

- **寺院の財政基盤を安定化**：定期的な収入源の確保と寄付の最適化
- **新たな檀家層の開拓**：地理的制約を超えた檀家獲得、若年層の取り込み
- **寺院文化の継承と革新**：伝統を守りながらも、現代的なコミュニティ運営を実現

### デジタル檀家証がもたらす新しい関係性

檀家証は単なる所属証明ではなく、寺院との関わり方を表現する「新しい絆のカタチ」です。

- **グレード例**：
  - 🌱 **萌芽（ほうが）**: 初めて寺院を支援する方向け
  - 🌿 **成長（せいちょう）**: 定期的に寺院を支援する方向け
  - 🌲 **繁栄（はんえい）**: 寺院の中核を担う檀家向け
  - 💎 **護持（ごじ）**: 寺院の主要な支援者・守り手向け

- **各グレードに応じた特典例**：
  - 特別法要への招待
  - 住職との直接対話の機会
  - 限定の寺院活動報告
  - 特別な祈願・回向の実施
  - 寺院運営の意思決定への参加権

## 利用ユーザ想定（ターゲット）

### 寺院・住職

- **ITリテラシーが高い若手住職**：デジタル化を積極的に取り入れたい30〜40代の住職
- **後継者不足・財政難に直面する寺院**：檀家減少や維持費増加に危機感を持つ寺院
- **地域活性化を目指す寺院**：観光や文化発信の拠点としての役割も担いたい寺院
- **革新的な取り組みに意欲的な寺院**：新しい檀家制度やコミュニティづくりに関心がある寺院

### 檀家や寺院支援者

- **透明性を重視する既存檀家**：寄付の使途をより明確に把握したい方
- **20〜30代の若年層**：従来の檀家制度に馴染みが薄いが、文化や寺院に関心がある若者
- **遠方・海外在住者**：地理的に寺院から離れているが、縁のある寺院を支援したい方
- **文化財保護・地域活性化に関心のある方**：日本の伝統文化や地域社会を支えたい方
- **ブロックチェーン技術に興味のある方**：社会貢献とテクノロジーの融合に関心を持つ層

## メリットや特徴

### 資金の透明化

- **ブロックチェーン上での資金管理**
  - 集まった寄付やお布施の使用履歴を記録・公開
  - 改ざん不可能な形で透明性を担保
  - 「このお金はどう使われたのか」が明確に

- **使途の明確化と報告**
  - 修繕や法要など、具体的な使途をあらかじめ提示
  - 実施後の報告も含めた「資金の一生」を可視化
  - 檀家との信頼関係構築に貢献

### 効率的な運営・業務改善

- **檀家台帳のデジタル化**
  - 紙ベースの台帳からの脱却
  - 情報検索・更新の効率化
  - 檀家とのコミュニケーション履歴も一元管理

- **寄付・お布施管理の自動化**
  - スマートコントラクト（キャニスター）による自動処理
  - 入金確認から礼状送付までの一連の流れを効率化
  - 寺院スタッフの業務負担軽減

- **法要予約・管理システム**
  - オンラインでの法要日程調整・予約機能
  - リマインダー自動送信やオンライン参加オプション
  - 檀家の利便性向上と寺院の予約管理効率化

### デジタル檀家証のグレード制

- **貢献度・関与度の可視化**
  - 檀家は自身の志向や可能な範囲でグレードを選択
  - 寺院への貢献が形として残り、やりがいにつながる
  - 次世代への継承もスムーズに

- **特典と権限の差別化**
  - グレードに応じた特別な体験や情報へのアクセス権
  - 優先参拝や限定イベントへの招待
  - 特別な祈願や回向の実施

### ガバナンス・DAO的運用

- **意思決定への参加**
  - グレードに応じて付与されるガバナンストークンで投票
  - 修繕計画や新しい取り組みへの意見表明
  - 檀家の声を反映した寺院運営の実現

- **コミュニティ形成**
  - オンライン上での檀家同士の交流の場
  - 寺院を中心とした新しいつながりの創出
  - 地理的制約を超えたコミュニティの形成

## ICPの仕組みをどの部分で使っているか

### 分散ID

- **ユーザー認証基盤**
  - Internet Identityを活用した安全なログイン
  - 個人情報保護とセキュアなアクセス管理
  - 柔軟な権限設定と管理

### Canisters

- **フロントエンド・バックエンド一体型のアプリケーション**
  - フロントエンドUIをキャニスターでホスティング
  - バックエンドロジックも同じくキャニスターで実装
  - サーバレスで運用可能な完全分散型アプリケーション

### StableMemory

- **データストレージ**
  - お寺の基本情報
  - 檀家データと関連情報
  - 寄付・お布施の履歴
  - グレード情報と特典履歴

## なぜICPの仕組みを活用して開発したか、その優位性や理由

### SNSによる分散型運営の可能性

- **Service Nervous System導入の展望**
  - 将来的にSNSを導入することでDAO的な運営が可能
  - 檀家と寺院のガバナンストークンによる共同意思決定
  - 真に分散化された寺院コミュニティの実現

### 低コスト（ガス代の安さ）

- **手数料の優位性**
  - Ethereum等の一般的なブロックチェーンに比べ、大幅に低コスト
  - 小額の寄付でも手数料に消えない経済合理性
  - 寺院の予算制約に適した設計

### ブラウザから直接アクセス可能

- **利用ハードルの低さ**
  - 特別なウォレットや拡張機能が不要
  - 通常のWebサービスと同様の使用感
  - 高齢檀家でも抵抗感なく利用できる設計

### 改ざん・ダウンリスクの低減

- **データの安全性と可用性**
  - 分散型ホスティングによる単一障害点の排除
  - データの永続性と耐障害性の確保
  - 重要な寺院記録や檀家情報の安全な保管

### 先行事例が少ないため差別化しやすい

- **先駆者優位の獲得**
  - ICPと寺院DXを組み合わせた事例はほぼ皆無
  - 伝統と最新技術の融合による話題性
  - 新しい市場の開拓と標準化を主導する可能性

### デプロイまでの開発スピード

- **効率的な開発プロセス**
  - 従来のクラウドやオンプレミスでの面倒な環境構築が不要
  - VPCやサーバー設定、リソース分析などの作業からの解放
  - アプリケーションロジックとUI開発に集中できる環境

### 運用の人的リソース削減

- **管理負荷の軽減**
  - サーバOS管理やパッチ適用などの運用作業が不要
  - セキュリティ対策の多くがプラットフォームレベルで提供
  - 寺院の少ない人的リソースでも継続的運用が可能

### 冗長構成の容易さ

- **システムの堅牢性**
  - 比較的低コストで高可用性を実現
  - 複数ノードでの自動レプリケーション
  - 災害時などのデータ喪失リスクを最小化

## 開発進捗と今後の展望

### 現在の開発状況

- **基本機能の実装完了**
  - 檀家登録・管理システム
  - グレード別檀家証発行機能
  - 寄付・お布施管理の基本機能
  - 寺院情報管理機能

- **StableMemoryでのデータ管理**
  - キャニスターのStableMemory（Key-Value方式）でデータを保持
  - 基本的なデータ構造とアクセス制御の実装

### 今後の開発計画

#### SNS(Service Nervous System)の導入

- **現状と課題**
  - 当初SNSの仕組みを使って運営方針の投票を行う想定だった
  - NNS（Network Nervous System）の承認が必要なため、現時点では導入を断念

- **将来展望**
  - リリース後の成長段階でSNSを導入し、本格的な分散型運営を実現
  - ガバナンストークンによる投票システムの構築
  - 檀家が直接寺院運営に参画できる仕組みの確立

#### トークンによる支払い機能

- **暗号資産決済の導入**
  - 寄付やお布施をICP、BTC、ETHなどで受け付ける機能
  - 国内法規制を考慮した適切な設計
  - 国際的な支援者からの寄付も容易に

#### StableMemoryからデータベースへの移行

- **現状の課題**
  - 現在はKey-Value方式でデータを保持しており、柔軟性に制限がある
  - 複数種類のデータ管理やスキーマ変更が発生すると拡張性やメンテナンス性に懸念

- **改善計画**
  - より柔軟なスキーマ管理・検索が可能なデータベース（CanDB）の検討
  - ただし、CanDBは1年以上更新されておらず、Motoko言語前提のため、Rust実装の可能性も探る
  - より安定した長期的なデータ管理ソリューションの模索

#### ユーザーインターフェースの改善

- **高齢檀家にも使いやすいUI/UX**
  - シンプルで直感的な操作性の追求
  - アクセシビリティ対応の強化
  - 多言語対応（英語・中国語など）で海外からの支援も促進

#### コミュニティ機能の拡充

- **檀家同士の交流促進**
  - メッセージングやグループ機能
  - 寺院行事の共同参加調整
  - 地域を超えたコミュニティ形成支援
